Information about this update

This update installs both the F24xx Flash Plug-In and the F28xx Flash Plug-In. It is equivalent to the v1.11.0 release with added support for the target connect/disconnect feature which exists in CCStudio 3.1 and newer. Note that this product will only work for CCStudio 3.1 and newer.

 

The remainder of this release notes is equivalent to that in v1.11.0.

 

There is only minor bug fix to the F24xx Flash Plug-In in this update. The update for the F28xx Flash Plug-In supplies the user with a new Flash28xx.ocx plug in file.

 

The new Flash28xx.ocx plug in has the following feature changes and bug fixes.

 

New Features

  • The following devices are now supported F2801, F2806 and F2808
  • The Flash Plug-In now automatically detects the device type. If the type is one of F2801, F2806 or F2808 then the correct device is automatically selected. If the type is one of F281x then the type is defaulted to F2812
  • The Depletion Recovery feature has been added for F2801, F2806 and F2808 devices only. This looks for sectors that are in depletion and attempts to recover them.
  • The Flash API Interface File field no longer has a default. When a File is selected, the plug-in will maintain this selection until the Device Type is changed at which point the file field will be blanked.

SDSsq35371

Flash plugin assumes path of d:\ti for clear, erase, program algorithms

C2000 On-chip Flash Programmer

C2400

If the default Clock Configuration in the On-Chip Flash Programmer plugin v1.10 for C2000 CCS 2.21 is changed, then when the clear, erase, program algorithms are recompiled, CCS assumes an install path of d:\ti. Since this may not be the correct install path, there will be build errors. The user needs to edit the path in the .pjt files before the projects can compile successfully.

SDSsq34046

Breakpoints in user code will prevent flash programming.

C2000 On-chip Flash Programmer

C2800

If you have breakpoints set in your code, you must disable all breakpoints before clicking the Execute Flash button. Failure to do so will result in a warning that your source line breakpoints could not be removed. The Flash API code will be loaded, but will not be executed because of this error.

SDSsq39603

Flash plugin cannot erase/program if breakpoints are set in CCS

C2000 On-chip Flash Programmer

C2800

The F28xx Flash Programmer plugin in CCS cannot erase/program into Flash if the existing code in Flash has breakpoints set.

SDSsq39111

Saving and restoring a Flash plugin workspace does not preserve settings

C2000 On-chip Flash Programmer

C2800

The frequency test settings in the C2000 On-chip Flash Programmer plugin are not correctly saved and restored as workspaces.

SDSsq34047

Placement of initialized data sections (like .const)

C2000 On-chip Flash Programmer

C2400

Initialized data sections (like .const) need to be placed in flash program space in order to be initialized correctly. The F24xx flash programmer is not responsible for initializing data space.  Refer to section 6.1.3 Allocating .const to Program Memory on page 6-5 of the TMS320C2x/C2xx/C5x Optimizing C Compiler User's Guide (SPRU024E) for details.  This manual is located in ..\docs\pdf directory under your Code Composer installation directory. F24xx Flash Plug-in.

SDSsq34229

F24xx flash plugin crashes when overinstalling the newer flash plugin

C2000 On-chip Flash Programmer

All

Installing the TMS320C2000 Code Composer Studio FLASH Plug-In v1.11 Flash plug-in patch over an existing flash plug-in installation results in initial failures of the F24x Flash plug-in v1.11.

Workaround:
If you have previously installed the TMS320C2000 Code Composer Studio FLASH Plug-In patch, you must uninstall it prior to installing the v1.11 update.

SDSsq34222

"Execution" displays confusing pop-up.

C2000 On-chip Flash Programmer

C2800

After hitting the "Execute Operation" button, a message appears asking the user to locate the file "FlashAPIInterface.c".  Note that this problem has been fixed for CCS 3.1.

Workaround:
The workaround for this problem is for the user to turn off the CCS option to look for source code when debugging as follows.

  • Select the "Option:Customize" menu item.
  • Under the Editor Properties tab, deselect the checkbox for the "Enable file browse while debugging" option.

SDSsq34227

Error msg appears twice when exiting CCS.

C2000 On-chip Flash Programmer

C2800

Upon exiting Code Composer Studio, the user may receive two warnings about not being able to remove breakpoints.

Workaround:
The workaround for this problem is to not turn on the option to remove all breakpoints when loading a new program. To do so:

  • Select the Options:Customize menu item.
  • Click on the Program Load Options tab.
  • Check the "Clear All Breakpoints When Loading New Programs" option.
  • Click "OK" to dismiss the Customize dialog box.

SDSsq36915

First two locations of OTP memory are not programmed correctly.

C2000 On-chip Flash Programmer

C2800

Using the CCS Flash Programmer plugin to program the F2812 OTP, the first two locations of the OTP are not programmed correctly.

 

Workaround:

If you encounter this problem (it does not always occur) then a workaround is to program the first two locations first and then program the rest
